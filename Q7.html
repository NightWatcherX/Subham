Q7.What is the main difference between batch processing and real-time processing in a transaction processing system?

Batch processing processes transactions as they occur, while real-time processing processes transactions at a scheduled time.
Batch processing groups similar transactions together and processes them at a scheduled time, while real-time processing processes transactions as they occur.
Batch processing processes transactions in a sequential order, while real-time processing processes transactions in a random order.

This is the Right answer.
Here are some information about this

A transaction processing system (TPS) is a type of information system that handles business transactions involving the collection, modification, and retrieval of data. A TPS can process transactions in two ways: batch processing or real-time processing. Batch processing groups similar transactions together and processes them at a scheduled time, while real-time processing processes transactions as they occur, without any delay. A TPS has four main components: inputs, outputs, storage, and processing system. Inputs are the requests for products or payments that come from outside parties, outputs are the documents or reports that the TPS generates after processing the inputs, storage is where the TPS keeps the data related to the transactions, and processing system is the software and hardware that perform the operations on the data. A TPS can offer several benefits to a business, such as increased transaction speed, improved cost efficiency, automated management, and enhanced reliability. Some examples of TPS are point of sale systems, online banking systems, inventory management systems, and payroll systems
